# Checklist Proyek Forum API: CI/CD & Security

---

## 1. Continuous Integration (CI)
- [ ] Workflow CI berjalan otomatis pada pull request ke branch utama (main/master)
- [ ] Menggunakan GitHub Actions
- [ ] Menjalankan semua test: unit, integration, functional
- [ ] Minimal ada 2 proses CI: 1 sukses, 1 gagal (bisa dibuktikan dengan screenshot)
- [ ] Menggunakan database test (container PostgreSQL di GitHub Actions atau database publik)
- [ ] Semua test dijalankan otomatis di pipeline
- [ ] Dokumentasi langkah CI di README

## 2. Continuous Deployment (CD)
- [ ] Workflow CD berjalan otomatis pada push ke branch utama (main/master)
- [ ] Menggunakan GitHub Actions (atau workflow lain)
- [ ] Deploy otomatis ke server (EC2, Vercel, Railway, Heroku, Render, dsb)
- [ ] Minimal ada 1 proses CD sukses (bisa dibuktikan dengan screenshot)
- [ ] Metode deploy sesuai platform (SSH, CodeDeploy, dsb)
- [ ] Dokumentasi langkah CD di README

## 3. Limit Access (Rate Limiting)
- [ ] Resource /threads dan seluruh path di dalamnya dibatasi
- [ ] Limit 90 request/menit per IP
- [ ] Konfigurasi NGINX diterapkan dan file nginx.conf dilampirkan di root proyek

## 4. HTTPS
- [ ] API hanya bisa diakses via HTTPS
- [ ] Domain/subdomain sendiri (misal: forumapi.dcdg.xyz)
- [ ] URL API HTTPS dicantumkan di student notes
- [ ] API lulus pengujian Postman via HTTPS
- [ ] SSL certificate terpasang di server

## 5. Kriteria Opsional (Nilai Plus)
- [ ] Fitur like/unlike komentar tersedia (PUT /threads/{threadId}/comments/{commentId}/likes)
- [ ] LikeCount muncul di setiap komentar pada response detail thread
- [ ] Fitur reply komentar tersedia
- [ ] Functional test untuk thread & comment ada
- [ ] Test coverage 100%
- [ ] Kode bersih dan konsisten (lint, style guide)

## 6. Pengiriman Submission
- [ ] Repo public & link repo + URL API HTTPS dicantumkan
- [ ] Folder proyek dalam ZIP, berisi package.json
- [ ] Tidak ada node_modules di dalam ZIP
- [ ] Tidak perlu melampirkan .env (pastikan variabel sudah sesuai)

---

**Tips:**
- Jalankan seluruh test sebelum submit
- Cek laporan coverage
- Review kode sebelum push (lint, format, dsb)
- Dokumentasikan semua langkah penting di README
